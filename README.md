# EvanGoldGrid - MT5 网格交易系统

## 概述

EvanGoldGrid 是一款专业的 MetaTrader 5 (MT5) 智能交易系统 (EA)，采用网格交易策略，支持多种交易模式和智能风险控制功能。适用于黄金 (XAUUSD) 及其他金融产品的自动化交易。

**版本**: 1.55  
**平台**: MetaTrader 5  
**语言**: MQL5  
**适用市场**: 外汇、贵金属、指数、加密货币等

## 核心特性

### 1. 四种交易模式

| 模式 | 代码值 | 说明 | 订单类型 |
|------|--------|------|----------|
| **做多 (Long)** | 0 | 所有网格均为买单 | BUY_LIMIT / BUY_STOP |
| **做空 (Short)** | 1 | 所有网格均为卖单 | SELL_LIMIT / SELL_STOP |
| **多空交替 (Alternating)** | 2 | 买单和卖单交替分布 | 混合订单 |
| **双边网格 (Dual Side)** | 3 | 每个网格同时有买单和卖单 | 双向订单 |

### 2. 智能网格系统

- **中心价格**: 可手动设置或实时刷新为当前市场价格
- **网格数量**: 自定义 (受最大订单数限制，默认 100)
- **网格间距**: 可自定义价格间隔 (点数)
- **自动补单**: 订单成交后自动在相同网格位置重新挂单
- **最大订单限制**: 防止超出经纪商限制 (默认 200)

### 3. 风险控制

- **动态净值止损**: 基于最高净值回撤百分比，触及止损线自动平仓撤单
- **移动止损**: 盈利后自动调整止损位锁定利润，支持保本止损和阶梯式追踪
- **订单数量限制**: 限制最大同时存在的订单数量
- **交易时段控制**: 可设置交易小时和允许的交易日

### 4. 交互式控制面板

实时显示交易状态，支持一键操作：
- 生成/重新生成网格
- 平亏损仓位
- 平盈利仓位
- 全部平仓
- 撤销所有挂单
- 撤销挂单 + 平仓
- 切换自动补单
- 切换盈利保护
- 切换移动止损
- 刷新中心价格

### 5. 智能网格移动 (v1.53)

- **自动跟随价格**: 当价格突破网格边界时自动移动网格
- **实时检查**: 每个 Tick 检查所有订单位置
- **智能判断**: 当订单价格偏离当前价格超过 2 倍间距时触发移动
- **自动重新挂单**: 移动后在新价格位置立即重新挂单
- **回测支持**: 回测时自动生成网格，无需手动操作

## 策略原理

### 网格交易基础

网格交易是一种在预设价格水平上建立头寸的策略。EA 会在中心价格上下等距离分布多个订单，形成"网格"。

```
价格分布示例 (中心价格=2000, 间距=1.0, 网格数=5):

网格 #0: 1998.0 ───────────────────────┐
网格 #1: 1999.0 ───────────────────┐   │
网格 #2: 2000.0 ───────────────┐   │   │  ← 中心价格
网格 #3: 2001.0 ───────────┐   │   │   │
网格 #4: 2002.0 ───────┐   │   │   │   │
                       │   │   │   │   │
当前价格：2000.5 ──────┘   │   │   │   │
                           │   │   │   │
订单类型：              SELL SELL BUY BUY
                       LIMIT LIMIT STOP STOP
```

### 订单类型判断逻辑

EA 根据网格价格与当前市价的关系自动选择订单类型：

**买单逻辑**:
- 网格价格 < 当前问价 (Ask) → **BUY_LIMIT** (限价买入，等待价格下跌成交)
- 网格价格 ≥ 当前问价 → **BUY_STOP** (止损买入，等待价格上涨成交)

**卖单逻辑**:
- 网格价格 > 当前报价 (Bid) → **SELL_LIMIT** (限价卖出，等待价格上涨成交)
- 网格价格 ≤ 当前报价 → **SELL_STOP** (止损卖出，等待价格下跌成交)

### 盈利机制

网格交易通过价格波动获利：
1. 价格在网格区间内震荡时，订单交替成交
2. 每个订单都有独立的止盈设置
3. 订单成交后，自动补单功能会在原位置重新挂单
4. 通过多次小盈利累积收益

1. **复制文件到 MT5 数据目录**:

- 盈利达到 1 倍网格间距时，启动保本止损（SL 移到开仓价 +0.5 点）
- 盈利达到 3 倍网格间距时，启动阶梯追踪（锁定 50% 未实现盈利）
- 止损位只朝有利方向移动，不会后退

## 交易逻辑详解

### 网格订单生成流程

```
1. 获取参数
   ├─→ 中心价格 (从编辑框或市价)
   ├─→ 网格数量
   ├─→ 网格间距
   ├─→ 止盈距离
   └─→ 手数

2. 计算网格价格
   ├─→ half_grids = grid_count / 2
   ├─→ grid_prices[i] = center_price + (i - half_grids) × grid_spacing
   └─→ 生成价格数组

3. 创建订单数组
   ├─→ 遍历每个网格价格
   ├─→ 根据模式确定订单方向
   ├─→ 根据价格与市价关系确定订单类型
   ├─→ 计算止损和止盈
   └─→ 填充订单信息结构

4. 批量下单
   ├─→ 每 10 个订单为一批
   ├─→ 使用 OrderSendAsync 异步发送
   ├─→ 批次间延迟 5ms
   └─→ 统计成功数量

5. 创建网格缓存
   ├─→ 记录每个网格的价格和方向
   ├─→ 用于后续补单和移动
   └─→ 标记缓存有效
```

### 自动补单机制

```
定时器触发 (每秒)
  │
  ├─→ 检查自动补单是否启用
  ├─→ 检查网格缓存是否有效
  ├─→ 检查是否在交易时间
  ├─→ 检查是否有人工干预
  │
  └─→ 遍历所有网格
       │
       ├─→ 检查网格是否有订单 (挂单或持仓)
       │
       ├─→ 如果无订单 → 补单
       │    ├─→ 确定订单类型 (LIMIT/STOP)
       │    ├─→ 计算止损止盈
       │    ├─→ 发送订单
       │    └─→ 更新显示
       │
       └─→ 如果有订单 → 跳过
```

### 盈利保护机制

```
每个定时器周期:
  │
  ├─→ 获取当前净值
  ├─→ 更新历史最高净值 (m_max_equity_ever)
  │
  ├─→ 计算止损线 = 最高净值 - 最大亏损额
  │
  └─→ 如果当前净值 < 止损线
       │
       ├─→ 触发保护
       ├─→ 撤销所有挂单
       ├─→ 关闭所有持仓
       ├─→ 停止交易标志
       └─→ 禁用自动补单
```

### 移动止损机制

```
每个 Tick:
  │
  ├─→ 检查移动止损是否启用
  ├─→ 遍历所有持仓
  │    │
  │    ├─→ 计算当前盈利点数
  │    │
  │    ├─→ 如果盈利 > 阈值 → 移动止损
  │    │   ├─→ 保本止损：盈利达到阈值后，SL 移到开仓价 + 微利
  │    │   └─→ 阶梯追踪：盈利每增加一定点数，SL 跟随移动
  │    │
  │    └─→ 发送止损更新请求
  │
  └─→ 遍历所有挂单
       └─→ 同样逻辑更新止损
```

**移动止损逻辑**:
- **保本止损**: 盈利达到 1 倍网格间距时，SL 移到开仓价 + 0.5 点
- **阶梯追踪**: 盈利继续增加时，SL 跟随移动，锁定 50% 的未实现盈利
- **只增不减**: 止损位只会朝有利方向移动，不会后退

### 网格移动机制

```
每个 Tick (如启用自动移动):
  │
  ├─→ 获取当前价格
  ├─→ 遍历所有网格
  │    │
  │    ├─→ 检查网格是否有订单
  │    │
  │    ├─→ 计算价格偏离度
  │    │   └─→ 如果偏离 > 2×间距 → 需要移动
  │    │
  │    └─→ 执行移动
  │         ├─→ 关闭原位置持仓
  │         ├─→ 删除原位置挂单
  │         ├─→ 在新价格重新挂单
  │         └─→ 更新网格缓存
  │
  └─→ 检查边界突破
       │
       ├─→ 如果当前价 > 上边界×(1+ 触发%) 
       │    └─→ 从底部移动网格到顶部
       │
       └─→ 如果当前价 < 下边界×(1- 触发%)
            └─→ 从顶部移动网格到底部
```

## 风险控制机制

### 1. 动态净值止损

```
初始化:
  m_initial_equity = 账户初始净值
  m_highest_equity = m_initial_equity  (运行期间最高净值)
  m_max_drawdown_percent = 用户设定的最大回撤百分比 (%)

每个 Tick 检查:
  current_equity = 当前净值
  
  如果 current_equity > m_highest_equity:
    → 更新 m_highest_equity = current_equity
    → 更新止损线 = m_highest_equity × (1 - m_max_drawdown_percent/100)
  
  如果 current_equity <= m_dynamic_stop_loss_line:
    → 触发动态止损
    → 调用 StopAllTrading()
    → 撤销所有挂单
    → 关闭所有持仓
    → 停止交易
    → 禁用自动补单
```

### 2. 移动止损（盈利锁定）

```
移动止损参数:
  m_profit_trailing_percent = 50.0  (盈利锁定百分比)
  m_grid_spacing = 网格间距

触发逻辑:
  ├─→ 保本止损：盈利 ≥ 1×网格间距
  │    └─→ SL = 开仓价 + 0.5 点
  │
  └─→ 阶梯追踪：盈利 ≥ 3×网格间距
       └─→ SL = 开仓价 + (盈利点数 - 2×间距)
       └─→ 锁定 50% 的未实现盈利
```

### 3. 订单数量限制

```
生成网格前:
  current_orders = CountMyOrders()
  available_slots = m_max_orders - current_orders
  max_grids = available_slots / 每网格订单数
  
  → 确保不超过最大订单数
```

### 4. 交易时间控制

```
IsTradingTime():
  ├─→ 检查是否为周日 (不交易)
  ├─→ 检查是否为周一 (如禁用则不交易)
  ├─→ 检查是否为周五 (如禁用则不交易)
  └─→ 检查是否在交易时段内
```

### 5. 人工干预检测

```
当用户执行以下操作时:
  ├─→ 平仓亏损/盈利/全部
  ├─→ 撤单
  └─→ 撤单 + 平仓
  
设置 m_manual_intervention = true
→ 暂停自动补单
→ 防止干扰用户操作
```

### 6. 移动止损详解

**工作原理**:
```
买单示例 (开仓价=2000, 间距=1.0):

价格涨到 2001 (盈利 100 点):
  → 触发保本止损
  → SL 从 0 移动到 2000.5

价格涨到 2003 (盈利 300 点):
  → 触发阶梯追踪
  → SL = 2000 + (300 - 200) = 2001.0
  → 锁定 100 点利润

价格涨到 2005 (盈利 500 点):
  → SL = 2000 + (500 - 200) = 2003.0
  → 锁定 300 点利润
```

**参数说明**:
- `InpProfitTrailingPercent`: 盈利锁定百分比 (默认 50%)
  - 值越大，止损跟进越慢，锁定利润越少
  - 值越小，止损跟进越快，锁定利润越多
  - 建议范围：30-70%

## 常见问题

### Q1: EA 不生成订单？

**检查清单**:
- [ ] 是否允许 EA 交易 (工具 → 选项 → EA)
- [ ] 图表右上角是否显示蓝色 EA 图标
- [ ] 是否在交易时间内
- [ ] 订单数量是否已达上限
- [ ] 账户余额是否足够
- [ ] 中心价格是否有效

### Q2: 自动补单不工作？

**检查清单**:
- [ ] 自动补单按钮是否显示"开"
- [ ] 是否进行了手动干预操作 (会暂停补单)
- [ ] 网格缓存是否有效
- [ ] 是否在交易时间

### Q3: 面板不显示？

**解决方案**:
- 回测模式下不显示面板 (正常)
- 实盘模式下重启 MT5
- 重新加载 EA
- 检查图表属性设置

### Q4: 订单被拒绝？

**可能原因**:
- 价格超出市场允许范围
- 手数不符合经纪商要求
- 账户类型不允许挂单
- 保证金不足
- 市场休市

### Q5: 如何查看日志？

**查看方法**:
- MT5 底部"专家"标签页 → EA 运行日志
- MT5 底部"日志"标签页 → 系统日志
- 关键操作都会输出详细日志

### Q6: 回测时如何设置参数？

**回测配置**:
1. 在 MT5 策略测试器中选择 EvanGoldGrid
2. 设置输入参数
3. 勾选"InpAutoStartGrid=true"自动生成网格
4. 运行回测
